Mission Brief: SQL Injection Analysis

Prerequisites:

  • Labtainer VM
  • Imodule Installed
    • The imodule for this lab is hosted here and can be installed with: 
    imodule https://github.com/tdanko128/imodule.io/raw/main/imodule.tar
    Please note that if you already have this imodule and are not able to find the lab please run: 
    imodule -u

Operation: SQL Injection Detection

Agent, your mission is to assess the security of Ethreal, a fictitious entity. Your objective is to test for SQL Injection vulnerabilities using SQLMap, analyze the reports, and recommend security improvements.

Mission Objectives

1. Identify Vulnerable Endpoints

  • Access the Ethreal website running on https://ethreal.com.
  • Identify input fields and parameters that may be vulnerable to SQL injection.

2. Run SQLMap

  • Execute SQLMap against identified endpoints to test for SQL Injection vulnerabilities.
  • Extract Data

3. Analyze Collected Data

  • Determine if the database is vulnerable to SQL injection.
  • Evaluate the database structure, if accessible.

4. File Your Intelligence Report

  • Provide a detailed analysis of findings.
  • Recommend defensive countermeasures Ethreal could implement to mitigate security risks.

Mission Execution: SQLMap Commands

Initialize the Lab

labtainer sqlmap

Visit the Target Site (Optional)

firefox https://ethreal.com

Explore SQLMap

Review the help documentation.

sqlmap --help

Run SQLMap Scan

Execute sqlmap against the target web application and extract database report. Complete the prompts

sqlmap --wizard
Please enter full target URL (-u): https://ethreal.com/backend/login
POST data (--data) [Enter for None]: Press `Enter`
Injection difficulty (--level/--risk). Please choose:
            [1] Normal (default)
            [2] Medium
            [3] Hard
        Choose `3`
Enumeration (--banner/--current-user/etc). Please choose:
            [1] Basic (default)
            [2] Intermediate
            [3] All
        Choose `3`

Intelligence Report: Analysis Questions

  • Was the site vulnerable to SQL injection? If so, what risks were identified?
  • What type of database management system (DBMS) was detected?
  • What countermeasures can mitigate SQL injection vulnerabilities?
  • What are the potential risks of a successful SQL injection attack?
  • Which SQL injection technique provided the most useful information?

Final Submission

Deliverables

  • SQLMap Report (HTML file).
  • Mission Analysis Report answering the questions above.
